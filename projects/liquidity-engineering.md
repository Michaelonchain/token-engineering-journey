# Liquidity Engineering

Liquidity engineering is one of the most important parts of token engineering.  
It focuses on how tokens can be traded smoothly without big price swings.

This file is part of my journey into understanding how liquidity works in Web3.

## ðŸ”¹ What Is Liquidity?
Liquidity means how easily a token can be bought or sold without changing its price too much.

A token with good liquidity:
- Trades smoothly  
- Has less volatility  
- Attracts investors  
- Supports healthy growth  

A token with weak liquidity:
- Easily pumps and dumps  
- Scares investors  
- Has unstable pricing

Most tokens today use AMMs like:
- Uniswap

- Orca  

These AMMs use formulas to set prices automatically.

The most common formula is:

Where:
- x = token A in the pool

## ðŸ”¹ Liquidity Pools
A liquidity pool usually contains:
- The token  
- A pairing asset like USDC, SOL, ETH, etc.

For example:

100,000 TOKEN + 20,000 USDC

This creates a price for the token and allows trading.
